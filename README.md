# Telegram-бот: автоматическая рассылка по группам

Асинхронный бот на **aiogram 3.x** для периодической рассылки сохранённых
постов во все группы, куда добавлен бот. Поддерживает `forward` (с
сохранением источника) и `copy` (без источника), произвольные подписи,
три режима ротации и гибкое расписание через APScheduler.

## Структура проекта

```
bot_rasssyl/
├── main.py              # точка входа
├── config.py            # загрузка .env
├── database.py          # async-обёртка над SQLite (aiosqlite)
├── broadcast.py         # сервис рассылки (анти-флуд, ретраи, ротация)
├── scheduler.py         # APScheduler AsyncIOScheduler
├── utils.py             # логирование, AdminFilter, классификация ошибок
├── handlers/
│   ├── __init__.py      # агрегатор роутеров
│   ├── posts.py         # /addpost, /listposts, /removepost, /clearposts, /setcaption
│   ├── groups.py        # /addgroup, /removegroup, /listgroups
│   ├── admin.py         # /setdelay, /setinterval, /setrotation, /sendnow, /status
│   └── common.py        # /start, /help, /id + catch-all
├── requirements.txt
├── .env.example
└── README.md
```

База данных (`bot.db`) и файл лога (`bot.log`) создаются автоматически
рядом с `main.py` при первом запуске.

## Установка

Нужен **Python 3.10+**.

```bash
# 1. Клонируем / разархивируем проект
cd bot_rasssyl

# 2. Создаём и активируем виртуальное окружение
python -m venv .venv
# Windows:
.venv\Scripts\activate
# Linux / macOS:
source .venv/bin/activate

# 3. Ставим зависимости
pip install -r requirements.txt

# 4. Готовим .env
cp .env.example .env     # Windows: copy .env.example .env
```

Отредактируйте `.env`:

```
BOT_TOKEN=123456:ABC-...            # токен от @BotFather
ADMIN_IDS=111111111,222222222       # ваш user_id (узнать через /id у @userinfobot)
DATABASE_PATH=bot.db
LOG_LEVEL=INFO
TIMEZONE=Europe/Moscow
```

Запуск:

```bash
python main.py
```

Остановка — `Ctrl+C`: бот корректно закроет планировщик, БД и сессию Telegram.

## Быстрый старт

### 1. Добавляем бота как администратора канала (опционально)

Если хотите использовать режим **forward** с показом исходного канала,
переслать сообщение из канала проще всего так:

1. Откройте нужный канал.
2. Выберите пост → «Переслать» → выберите свой приватный чат с ботом.
3. В приватном чате ответьте на это сообщение командой:
   ```
   /addpost forward
   ```

Для режима **copy** (без указания источника):

```
/addpost            (по умолчанию copy)
/addpost copy       (то же явно)
```

### 2. Добавляем группы

1. Добавьте бота в нужную группу как участника (если группа разрешает
   писать обычным участникам — этого достаточно; иначе сделайте админом).
2. В самой группе напишите:
   ```
   /addgroup
   ```
   Бот проверит, что может писать, и добавит группу в базу.

Повторите для каждой группы. Проверить список:

```
/listgroups
```

### 3. Настраиваем расписание

**Пример задачи:** рассылать раз в 6 часов, задержка между группами
20 секунд, по кругу (round-robin).

```
/setinterval 6h          # интервал между циклами
/setdelay 20             # задержка между группами в одном цикле
/setrotation round       # по кругу
```

Проверяем:

```
/status
```

Увидите что-то вроде:

```
📊 Статус бота
• Активных групп: 12
• Сохранённых постов: 3
• Задержка между группами: 20 сек
• Интервал между циклами: 360 мин (~6.0 ч)
• Режим ротации: round
• Последний цикл: никогда
• Следующий цикл: через 5 ч 59 мин (…)
```

Хотите прогнать прямо сейчас, не дожидаясь цикла?

```
/sendnow          # рассылает следующий по ротации пост
/sendnow 2        # рассылает конкретный пост #2
```

## Команды целиком

### Посты (приватный чат с ботом)

| Команда | Что делает |
|---|---|
| `/addpost [forward\|copy]` (реплай) | Сохранить сообщение; по умолчанию `copy` |
| `/listposts` | Список сохранённых постов |
| `/removepost ID` | Удалить пост |
| `/clearposts` | Удалить все посты |
| `/setcaption ID top\|bottom текст` | Добавить/заменить подпись сверху или снизу |
| `/setcaption ID none` | Снять подпись |

### Группы

| Команда | Где вызывать | Что делает |
|---|---|---|
| `/addgroup` | в группе | добавить текущую группу |
| `/removegroup` | в группе | удалить текущую |
| `/removegroup -100…` | в ЛС с ботом | удалить по chat_id |
| `/listgroups` | в ЛС | список всех групп |

### Расписание и поведение

| Команда | Пример | Описание |
|---|---|---|
| `/setdelay N` | `/setdelay 20` | задержка между группами, сек |
| `/setinterval X` | `/setinterval 6h` / `30m` / `1d` / `240` (мин) | интервал циклов |
| `/setrotation round` | — | пост за постом по кругу |
| `/setrotation random` | — | случайный выбор |
| `/setrotation single ID` | `/setrotation single 5` | всегда один пост |
| `/sendnow [ID]` | `/sendnow` | запустить цикл сразу |
| `/status` | — | сводка состояния |

### Прочее

| Команда | Что делает |
|---|---|
| `/start`, `/help` | справка |
| `/id` | показать ваш user_id и chat_id (удобно для настройки) |

## Безопасные значения задержек (реальные лимиты Telegram)

Telegram не публикует жёсткие цифры, но документированно действуют
такие ориентиры:

* ~**30 сообщений в секунду** на весь бот — глобальный потолок.
* **1 сообщение в секунду** на один чат в долгой перспективе.
* **20 сообщений в минуту** на одну группу, если шлёте подряд.

Поэтому рекомендуется:

| Сценарий | `/setdelay` | `/setinterval` | Ротация |
|---|---|---|---|
| До 20 групп, мягкий режим | **20–30 сек** | `6h` – `12h` | round |
| 20–100 групп | **15–20 сек** | `6h` | round |
| 100+ групп | **10–15 сек** + свежий токен | `8h` – `12h` | round/random |
| Стресс-тест | не ниже **8 сек** | `12h+` | — |

Дополнительно бот уже соблюдает:

* **Cooldown 5 минут** на одну и ту же группу — если `/sendnow` был только
  что, следующий цикл пропустит эту группу, чтобы не ловить FloodWait.
* **Обработка `TelegramRetryAfter`** — бот честно ждёт столько, сколько
  запросил Telegram, и делает одну повторную попытку.
* **Авто-деактивация** группы при `Forbidden` / `bot was kicked` /
  `chat not found` — чтобы мёртвые чаты не ломали весь цикл.
* **Очередь**: два цикла никогда не идут параллельно (`asyncio.Lock`).

## Пример: «рассылать каждые 6 часов с задержкой 20 сек»

1. Запустите бота и откройте с ним личный чат, нажмите `/start`.
2. Перешлите боту нужный пост из канала.
3. Ответьте на него:
   ```
   /addpost forward
   ```
   (`copy`, если не хотите показывать источник).
4. Опционально — подпись под постом:
   ```
   /setcaption 1 bottom Подписывайтесь на канал: @mychannel
   ```
5. Добавьте бота в каждую группу, в группе:
   ```
   /addgroup
   ```
6. В ЛС настройте расписание:
   ```
   /setdelay 20
   /setinterval 6h
   /setrotation round
   ```
7. Проверьте: `/status`. Запустите «тестовый» прогон: `/sendnow`.
8. Дальше бот будет сам запускать цикл каждые 6 часов.

## Логирование

Все события (добавление постов, отправки, деактивация групп, FloodWait,
ошибки) пишутся в два места:

* **stdout** — полезно, если запускаете под systemd / Docker.
* **bot.log** — ротируется по 5 МБ, хранит 3 архивных файла.

Формат строки:

```
2026-04-23 10:00:00,123 | INFO    | broadcast | Цикл завершён: post=1 sent=12 …
```

## Запуск как сервис (systemd, Linux)

`/etc/systemd/system/tg-broadcast.service`:

```ini
[Unit]
Description=Telegram broadcast bot
After=network-online.target

[Service]
Type=simple
User=botuser
WorkingDirectory=/opt/bot_rasssyl
ExecStart=/opt/bot_rasssyl/.venv/bin/python main.py
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now tg-broadcast.service
journalctl -u tg-broadcast.service -f
```

## Частые вопросы

**Почему бот показывает «Переслано от вашего имени» в режиме forward?**
Если сообщение в приватном чате с ботом вы набрали САМИ, Telegram не
знает о вышестоящем источнике. Пришлите нужный пост **пересылкой** из
канала — тогда цепочка «Переслано от канала X» сохранится.

**Работает ли бот с приватными каналами?**
С форвардом из приватного канала работает, если вы сначала пересылаете
пост в приватный чат с ботом. Внутри сохранённой цепочки Telegram
покажет «Переслано из частного канала» (без ссылки).

**Можно ли использовать MarkdownV2?**
Да — по умолчанию используется HTML (проще экранировать),
но в `utils.escape_markdown_v2()` есть хелпер. При отправке через
`bot.send_message(..., parse_mode=ParseMode.MARKDOWN_V2)` вы получаете
поддержку MarkdownV2 точечно.

**Что если Telegram отправил `Too Many Requests: retry after 120`?**
Бот сам ждёт 120 секунд + 1 секунда запаса и делает одну повторную
попытку. Если и она неудачна — группа отмечается ошибочной, но цикл
не падает целиком. Рассмотрите увеличение `/setdelay`.
